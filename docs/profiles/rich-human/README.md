# Rich-human profile
Use this profile for human-first documentation that benefits from structure.
It applies the full Structured Richness rules and language checks.
Choose it when clarity and onboarding speed matter most.
This profile is the default when no frontmatter is present.

## When to use
This profile is designed for primary, narrative documentation.
Use it when readers need a clear overview and reasoning.

- README files.
- Design docs.
- Concept docs.
- Onboarding guides.

## What it enforces
The profile enforces the complete Structured Richness rule set.

- One H1 and sentence-case headings.
- H3 maximum heading depth.
- Callouts allowed within the spec limits.
- Vale language checks enabled.
- Markdownlint rules enforced.

## What it relaxes
No additional relaxations beyond the base spec.

- None.

## Why it exists
Rich-human ensures core docs are easy to scan and understand.
It balances structure and readability without extra process overhead.

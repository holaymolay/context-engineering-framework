# Internal-notes profile
Use this profile for personal notes and early drafts.
It relaxes structure to keep exploratory work lightweight.
This profile is not for published or shared documentation.
Its purpose is to avoid bypassing the documentation system.

## When to use
This profile is intended for private, temporary writing.

- Personal notes.
- Scratch docs.
- Early drafts.
- Exploratory writing.

## What it enforces
It keeps only the lightest structure rules.

- One H1 and ordered heading levels.
- H4 maximum heading depth.
- Markdownlint relaxed.

## What it relaxes
It relaxes language and callout constraints.

- No Vale enforcement.
- Callouts allowed but not enforced.
- Longer paragraphs allowed.
- No CI enforcement by default.

## Why it exists
Internal-notes keeps low-stakes writing inside the system.
It prevents pressure to bypass documentation rules entirely.
